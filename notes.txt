## 1) Define the API contract

Create `docs/api-contract.md` with the three dashboard needs per authority:

**Autopilot prompt:**

Write `docs/api-contract.md` defining a minimal consistent contract for authority dashboards:

1. **GET /api/observations/count** → `{count: number}`
   - Optional query param: `authority` (string)
   
2. **GET /api/observations/recent?limit=5** → `[{postcode, measurements, observation, createdAt, contributorId}]`
   - Query params: `limit` (number, default 5, max 50), optional `authority` (string)
   - Response array ordered by createdAt DESC
   
3. **GET /api/rewards/leaderboard?limit=3** → `[{contributorId, points, rank}]`
   - Query params: `limit` (number, default 3, max 50), optional `authority` (string)
   - Response array ordered by points DESC

Include error response shape:
```json
{
  "timestamp": "ISO8601 string",
  "status": 400|404|500|503,
  "error": "Error type",
  "message": "Human-readable message",
  "path": "/api/observations/count"
}
```

Status codes:
- 200 OK - Success
- 400 Bad Request - Invalid query params
- 404 Not Found - Resource not found
- 500 Internal Server Error - Server error
- 503 Service Unavailable - Downstream service unavailable

Keep it simple and implementation-agnostic.

**Gate:** Contract file exists and is readable.

```json
{
  "timestamp": "ISO8601",

}
```

## 3) Standardize authority services (minimal adapters)

Goal: Each authority folder must expose the contract endpoints used by React, even if internally it calls existing microservices.

### 3.1 Add a lightweight "API adapter" per authority

**Autopilot prompt:**

For each authority (authority-a, authority-b, authority-c, authority-d), implement/ensure the API contract endpoints exist:

- **GET /api/observations/count** (optional `?authority=X`)
- **GET /api/observations/recent?limit=5** (optional `?authority=X`)
- **GET /api/rewards/leaderboard?limit=3** (optional `?authority=X`)

**Implementation strategy per authority:**

**Authority-A (CitizenScienceWaterQualityApplication):**
- This is a monolithic app on port 8080
- Add adapter endpoints that:
  - Map `/api/observations/count` → query `CrowdsourcedRepository.count()`
  - Map `/api/observations/recent?limit=5` → query `CrowdsourcedRepository.findAll()` ordered by creation time, limit 5
  - Map `/api/rewards/leaderboard?limit=3` → query `RewardService` to get top 3 contributors by points
- Add a new `AdapterController` class with these 3 endpoints
- Do NOT modify existing controllers, only add new ones

**Authority-B (water-quality-monitoringfinal):**
- Has api-gateway (8080), crowdsourced-data-service (8081), rewards-service (8082)
- Add adapter endpoints in the api-gateway that:
  - Map `/api/observations/count` → call crowdsourced-data-service `/api/data/stats` and extract total count
  - Map `/api/observations/recent?limit=5` → call crowdsourced-data-service `/api/data/observations?limit=5` and transform response
  - Map `/api/rewards/leaderboard?limit=3` → call rewards-service `/api/rewards/leaderboard?limit=3` and transform response
- Add adapter controller in api-gateway that proxies to internal services and transforms responses

**Authority-C (water-quality-service):**
- Has api-gateway (8080), data-service (8081), rewards-service (8082)
- Add adapter endpoints in the api-gateway that:
  - Map `/api/observations/count` → call data-service `/api/data/observations/count?authority=X`
  - Map `/api/observations/recent?limit=5` → call data-service `/api/data/observations/latest?limit=5&authority=X` and transform response
  - Map `/api/rewards/leaderboard?limit=3` → call rewards-service `/api/rewards/leaderboard?authority=X&limit=3` and transform response
- Add adapter controller in api-gateway

**Authority-D (WaterQuality):**
- Has Gateway (8080), CrowdsourcedDataService (8081), RewardsService (8082), uses MongoDB
- Add adapter endpoints in the Gateway that:
  - Map `/api/observations/count` → call CrowdsourcedDataService `/api/observations` (list all) and count, or add a count endpoint
  - Map `/api/observations/recent?limit=5` → call CrowdsourcedDataService `/api/observations` and transform to contract format, limit 5
  - Map `/api/rewards/leaderboard?limit=3` → call RewardsService to get rewards for all citizens, sort by points, return top 3 (may need to add leaderboard endpoint)
- Add adapter controller in Gateway

**Important:**
- Do NOT modify existing microservice code. Only add new adapter controllers/endpoints.
- If existing services use different paths/fields, add an adapter layer to map to the contract response shapes.
- Include basic input validation and error handling.
- If a downstream dependency is down, return 503 with proper error JSON.

**Gate:** Each authority responds correctly to the 3 endpoints. `limit` works. Service returns 503 if downstream dependency is down.

These instructions account for the actual structure and differences of the four microservices.
